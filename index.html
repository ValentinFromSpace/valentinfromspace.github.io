IC & IPFS

This website is hosted on IPFS, the interplanetary file system. IPFS is a decentralized peer-to-peer network. Files are addressed via cryptographic hashes (content identifier or CID) rather than URLs. The downside of URLs is that they tell you where exactly something is located; CIDs on the other hand tell you what something is. Suppose someone tells you about a book. The other person didn't tell you the title of the book, but rather in which library you'll find it, and on exactly which shelf and row it is located. The library however is located on the other end of the planet and it would take you a long time to get there. And once you get there, you notice the library was renovated and the book is somewhere else entirely.

That's not how we communicate in real life, and it shouldn't be how we communicate on the web. Just like the other person tells you the title of the book so you can choose where you want to get the book yourself, you tell IPFS the hash of the file you want and the network will find the closest node that has your file. This happens to be a lot faster than the old way.

It is likely your browser got this website through a public IPFS gateway (in this case cloudflare-ipfs.com). If you instead want to be part of the the IPFS peer-to-peer network, visit my website on the brave browser and enable its native IPFS integration. Brave will then download an IPFS node to your device. You can now fetch this website directly from my node.

I'm also looking to get this website running on the internet computer. The IC is a web of globally distributed nodes that execute "canisters". A canister is a smart-contract, but much more capable. Canisters include both code and persistent state.
